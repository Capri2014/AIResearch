## Pipeline PR #2 (This PR)

**Focus**: Temporal Waypoint BC → CARLA Integration

### What was done

Created temporal waypoint policy wrapper for CARLA closed-loop evaluation:

**Files created:**
1. `sim/driving/carla_srunner/temporal_policy_wrapper.py`
   - `TemporalEncoder`: Full CNN+LSTM encoder for temporal waypoint prediction
   - `TemporalWaypointPolicyWrapper`: CARLA integration wrapper
   - Methods: `predict_temporal()`, `add_frame()`, `predict_and_control()`
   - Supports checkpoint loading with various state_dict formats

2. `training/eval/run_temporal_carla_eval.py`
   - CARLA closed-loop evaluation for temporal waypoint BC
   - `TemporalWaypointAgent`: Maintains frame buffer for temporal context
   - Metrics: ADE, FDE, route_completion, temporal_consistency
   - 5 scenarios: straight_clear, straight_cloudy, straight_night, straight_rain, turn_clear

### Key components

1. **TemporalEncoder** (temporal_policy_wrapper.py):
   - CNN backbone (resnet18/resnet34/efficientnet_b0)
   - LSTM for temporal aggregation
   - Waypoint head for final predictions
   - Handles (B, T, C, H, W) input sequences

2. **TemporalWaypointPolicyWrapper**:
   - Frame buffer management
   - Checkpoint loading with flexible state_dict handling
   - Waypoints to CARLA control conversion
   - Real-time inference via `predict_and_control()`

3. **TemporalEvalMetrics**:
   - Core: route_completion, ADE, FDE, success_rate
   - Temporal-specific: temporal_consistency
   - Safety: collisions, red_light_violations
   - Performance: avg_inference_time_ms

### Usage

```bash
# Train temporal waypoint BC
python -m training.sft.train_temporal_waypoint_bc \
  --episodes-glob "out/episodes/**/*.json" \
  --sequence-length 4 \
  --batch-size 16 \
  --out-dir out/temporal_waypoint_bc

# Evaluate in CARLA
python -m training.eval.run_temporal_carla_eval \
  --checkpoint out/temporal_waypoint_bc/best_model.pt \
  --output-dir out/temporal_carla_eval \
  --sequence-length 4
```

## Pipeline PR #3 (This PR)

**Focus**: SSL-to-Waypoint BC Transfer Learning

### What was done

Created transfer learning pipeline that bridges **SSL pretrain → waypoint BC**:

**File created:** `training/sft/train_ssl_to_waypoint_bc.py`

**Classes:**
- `SSLEncoder`: Loads pretrained SSL checkpoints (contrastive/JEPA)
- `TemporalSSLEncoder`: CNN + LSTM for temporal sequences
- `SSLToWaypointBC`: Full model with encoder + waypoint head

**Key features:**
- Loads pretrained checkpoints from contrastive, temporal contrastive, JEPA training
- Supports frozen encoder (transfer learning) or fine-tuning mode
- Compatible with resnet18, resnet34, efficientnet_b0 backbones
- Handles both single-frame and temporal sequence inputs
- Built-in evaluation with ADE/FDE metrics

### Usage

```bash
# Transfer learning (frozen encoder)
python -m training.sft.train_ssl_to_waypoint_bc \
  --train-episodes "out/episodes/**/*.json" \
  --pretrained-path out/pretrain/contrastive/best_model.pt \
  --backbone resnet18 \
  --freeze-encoder \
  --batch-size 32 \
  --out-dir out/ssl_waypoint_bc

# Fine-tuning
python -m training.sft.train_ssl_to_waypoint_bc \
  --train-episodes "out/episodes/**/*.json" \
  --pretrained-path out/pretrain/temporal/best_model.pt \
  --use-temporal \
  --no-freeze \
  --out-dir out/ssl_waypoint_bc_ft
```

## Pipeline PR #5 (This PR - 4:30pm PT)

**Focus**: Kinematic Waypoint Environment for RL Refinement (Option B)

### What was done

Created a new kinematic toy waypoint environment for RL refinement after SFT:

**File created:** `training/rl/kinematic_waypoint_env.py`

**Classes:**
- `KinematicCar`: Bicycle model for realistic car kinematics
- `KinematicWaypointEnv`: Environment that consumes predicted waypoints
- `SFTWaypointLoader`: Loads SFT waypoint model checkpoints
- `DeltaWaypointPolicy`: Delta head for residual learning (Option B)
- `ValueFunction`: Value function for PPO

**Key features:**
1. **Bicycle model kinematics**: Forward velocity, steering angle affecting heading
2. **Waypoint consumption**: Environment follows predicted waypoints using kinematic control
3. **Option B (Delta Waypoints)**: `final_waypoints = sft_waypoints + delta_head(z)`
4. **SFT checkpoint loading**: Can initialize from pretrained SFT waypoint model
5. **Proper metrics output**: Saves to `out/kinematic_rl/<run_id>/metrics.json` and `train_metrics.json`

### Design Pattern (Option B)

```
final_waypoints = sft_waypoints + delta_head(z)
```

Where:
- `sft_waypoints`: Predictions from frozen SFT model
- `delta_head(z)`: Small trainable network predicting corrections
- `z`: Latent state encoding

This achieves:
1. **Sample efficiency**: Only trains a small delta head (not full model)
2. **Safety**: Corrections are bounded, preserving SFT's reasonable behavior
3. **Modularity**: Delta head can be swapped/updated independently

### Usage

```bash
# Run smoke test
python -m training.rl.kinematic_waypoint_env \
  --out-dir out/kinematic_rl \
  --episodes 50

# With SFT checkpoint
python -m training.rl.kinematic_waypoint_env \
  --sft-checkpoint out/sft_waypoint_bc/model.pt \
  --out-dir out/kinematic_rl \
  --episodes 200
```

