# Daily Notes - 2026-03-13

## Pipeline PR #2: End-to-End Inference Runner

### Created: `training/eval/run_e2e_inference.py`
- **E2EInferenceConfig**: Configuration dataclass for end-to-end inference
- **E2EInferenceRunner**: Main runner class that loads checkpoint and runs evaluation
- **E2EInferenceRunner.run()**: Run evaluation across multiple episodes
- **E2EInferenceRunner.run_episode()**: Single episode evaluation
- **E2EInferenceRunner.predict_waypoints()**: Bridge to WaypointInference
- **E2EInferenceRunner.compute_control()**: Bridge to WaypointTrackingController
- Auto-detects latest BC/RL checkpoints
- Mock mode for testing without CARLA
- Outputs schema-compliant metrics.json

### Features:
- **Multi-policy support**: BC, RL, SFT+Delta checkpoints
- **Auto-detection**: Finds latest checkpoint if not provided
- **Controller integration**: Uses WaypointTrackingController for control
- **Mock environment**: For testing without CARLA
- **CARLA-ready**: --use-carla flag for real evaluation

### Architecture:
```
Checkpoint (BC/RL/SFT_Delta)
    ↓
E2EInferenceRunner
    ├── WaypointInference (from checkpoint)
    │       ↓
    │   waypoints [num_waypoints, 2]
    │       ↓
    ├── WaypointTrackingController
    │       ↓
    │   control (throttle, steer, brake)
    │       ↓
    └── Episode evaluation → metrics.json
```

### Usage:
```bash
# Dry-run (validate config)
python3 -m training.eval.run_e2e_inference --dry-run

# Run with auto-detected checkpoint (mock)
python3 -m training.eval.run_e2e_inference --episodes 10

# Run with specific checkpoint
python3 -m training.eval.run_e2e_inference \
    --checkpoint out/waypoint_bc/run_20260312_083423/best.pt \
    --episodes 10

# Run with RL checkpoint
python3 -m training.eval.run_e2e_inference \
    --policy-type rl \
    --episodes 10

# Real CARLA evaluation (if available)
python3 -m training.eval.run_e2e_inference \
    --use-carla \
    --carla-host 127.0.0.1 \
    --episodes 10
```

### Output:
- `out/e2e_inference/metrics_<run_id>.json`
- Schema-compliant with `data/schema/metrics.json`
