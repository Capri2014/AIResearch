## Pipeline PR #1: Proper SFT + RL Training Pipeline

### Summary
Fixed the fundamental issue with RL after SFT training. The previous implementation had a randomly initialized SFT model as baseline, causing the delta head to learn meaningless corrections (0% goal rate). This PR implements a proper two-stage pipeline:
1. Stage 1: Train SFT model via supervised learning (behavior cloning)
2. Stage 2: Train residual delta head on top of frozen SFT via RL

### Root Cause Analysis
Previous RL training showed 0% goal rate because:
- `PPOResidualWaypointAgent` used a randomly initialized `SFTWaypointModel` as baseline
- The evaluation script used `env.get_sft_waypoints()` (linear interpolation) as SFT baseline
- This mismatch caused the delta head to learn useless corrections

### Changes

#### `training/rl/proper_sft_rl_pipeline.py`
New file implementing proper two-stage training:
- **Stage 1: SFT Training**
  - `WaypointDataset`: Generates state-waypoint pairs from environment
  - `SFTWaypointPredictor`: MLP that learns waypoint prediction via supervised learning
  - `train_sft_model()`: Trains SFT model with MSE loss
  
- **Stage 2: RL Training**
  - `ResidualDeltaHead`: Predicts delta adjustments to SFT waypoints
  - `ValueFunction`: Value estimator for PPO
  - `ProperPPOAgent`: PPO with proper residual learning
  - `train_rl_residual()`: Trains delta head while keeping SFT frozen
  
- **Evaluation**
  - `evaluate_agent()`: Computes ADE/FDE metrics

#### `training/rl/load_checkpoint.py`
New utility for loading checkpoints:
- `load_sft_checkpoint()`: Load SFT model only
- `load_full_checkpoint()`: Load SFT + delta head + value fn
- `get_policy_fn()`: Get a policy function from checkpoint

### Key Insight
RL can only improve upon a REASONABLE baseline. The SFT model must be trained first (via supervised learning) before RL can learn useful corrections. The residual formula: `final_waypoints = sft_waypoints + delta_head(state)` only works when `sft_waypoints` is a decent predictor.
