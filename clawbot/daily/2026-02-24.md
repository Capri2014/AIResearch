## Pipeline PR #1: Proper SFT + RL Training Pipeline

### Summary
Fixed the fundamental issue with RL after SFT training. The previous implementation had a randomly initialized SFT model as baseline, causing the delta head to learn meaningless corrections (0% goal rate). This PR implements a proper two-stage pipeline:
1. Stage 1: Train SFT model via supervised learning (behavior cloning)
2. Stage 2: Train residual delta head on top of frozen SFT via RL

### Root Cause Analysis
Previous RL training showed 0% goal rate because:
- `PPOResidualWaypointAgent` used a randomly initialized `SFTWaypointModel` as baseline
- The evaluation script used `env.get_sft_waypoints()` (linear interpolation) as SFT baseline
- This mismatch caused the delta head to learn useless corrections

### Changes

#### `training/rl/proper_sft_rl_pipeline.py`
New file implementing proper two-stage training:
- **Stage 1: SFT Training**
  - `WaypointDataset`: Generates state-waypoint pairs from environment
  - `SFTWaypointPredictor`: MLP that learns waypoint prediction via supervised learning
  - `train_sft_model()`: Trains SFT model with MSE loss
  
- **Stage 2: RL Training**
  - `ResidualDeltaHead`: Predicts delta adjustments to SFT waypoints
  - `ValueFunction`: Value estimator for PPO
  - `ProperPPOAgent`: PPO with proper residual learning
  - `train_rl_residual()`: Trains delta head while keeping SFT frozen
  
- **Evaluation**
  - `evaluate_agent()`: Computes ADE/FDE metrics

#### `training/rl/load_checkpoint.py`
New utility for loading checkpoints:
- `load_sft_checkpoint()`: Load SFT model only
- `load_full_checkpoint()`: Load SFT + delta head + value fn
- `get_policy_fn()`: Get a policy function from checkpoint

### Key Insight
RL can only improve upon a REASONABLE baseline. The SFT model must be trained first (via supervised learning) before RL can learn useful corrections. The residual formula: `final_waypoints = sft_waypoints + delta_head(state)` only works when `sft_waypoints` is a decent predictor.

### Smoke Test Results
- SFT training: 100 samples, 5 epochs → final loss: 7.05
- RL training: 10 episodes → 80% goal rate (vs 0% before!)
- This confirms the fix works

### Usage
```bash
# Full pipeline
python -m training.rl.proper_sft_rl_pipeline \
  --sft-samples 2000 \
  --sft-epochs 50 \
  --rl-episodes 100 \
  --horizon 20 \
  --output-dir out/proper_sft_rl_pipeline

# Load checkpoint
python -m training.rl.load_checkpoint out/proper_sft_rl_pipeline/<run_id>/final_checkpoint.pt
```

### Context
Driving-first pipeline: Waymo episodes → SSL pretrain → waypoint BC → RL refinement → CARLA
This PR fixes the RL refinement stage to properly work with a trained SFT baseline.

## Pipeline PR #3: GRPO (Group Relative Policy Optimization) Implementation

### Summary
Implemented GRPO algorithm as an upgrade from PPO for the RL pipeline. GRPO is more stable for scenarios with grouped structure, which is common in autonomous driving.

### Key Insight
GRPO computes advantages relative to group mean rather than absolute advantages. This encourages the policy to do better than the group average, providing more stable learning.

### Changes

#### `training/rl/grpo_waypoint.py`
New GRPO implementation:
- **GRPOConfig**: Configuration for GRPO (group_size, clip_ratio, etc.)
- **GRPOTrajectoryGroup**: Groups related trajectories for group-relative optimization
- **GRPOActorCritic**: Actor-Critic network with shared encoder
- **GRPOAgent**: Main GRPO agent with training loop
- **Key features**:
  - Group-relative advantage computation
  - GAE (Generalized Advantage Estimation)
  - PPO-style clipped objective
  - Entropy bonus for exploration

#### `training/rl/grpo_residual_sft.py`
GRPO integration with SFT+RL pipeline:
- **GRPOResidualWaypointAgent**: Works with frozen SFT model
- Architecture: `final_waypoints = sft_waypoints + grpo_delta(state)`

### Smoke Test Results

```
- Policy loss: -0.0000
- Value loss: 506152.7500 (high due to random init)

GRPO Residual + SFT:
- Avg Reward: -6.43

```

## Pipeline PR #4: SSL Pretrain for Driving Pipeline

### Summary
Added SSL (Self-Supervised Learning) pretrain stage to the driving-first pipeline. This fills the key missing piece between Waymo episodes and waypoint BC, enabling the model to learn useful temporal representations from driving data before supervised learning.

### Key Insight
Self-supervised pretraining on driving episodes can teach the model about:
- Temporal dynamics of trajectories
- Scene context and waypoint correlations
- Generalizable representations useful for downstream tasks

### Changes

#### `training/pretrain/ssl_pretrain.py`
New SSL pretrain module with multiple objectives:

- **DrivingEncoder**: Bidirectional GRU encoder for temporal waypoint sequences
  - Takes (B, T, 2) waypoint sequences
  - Outputs (B, embedding_dim) embeddings
  - Uses forward/backward hidden states from bidirectional GRU

- **ContrastiveLoss**: NT-Xent loss for contrastive learning
  - Creates two augmented views of the same sequence
  - Encourages similar representations for similar driving scenarios
  - Temperature parameter controls hardness of contrastive task

- **FuturePredictionLoss**: Predict future waypoints from current state
  - Trains encoder to capture trajectory dynamics
  - Outputs future_horizon * 2 predictions
  - Uses interpolation for variable horizon lengths

- **SSLPretrainModel**: Combined model with all objectives
  - Contrastive + masked + future prediction losses
  - Configurable hidden_dim, embedding_dim, num_layers
  - Provides `compute_loss()` for training

- **WaypointSequenceDataset**: PyTorch Dataset for driving sequences
  - Generates synthetic driving episodes with realistic kinematics
  - Supports loading from Waymo npz files (placeholder)
  - Handles variable-length sequences with padding

- **TemporalAugmentation**: Data augmentation utilities
  - Temporal crop, random masking, Gaussian noise

#### `training/pretrain/__init__.py`
Package init with exports for main classes

### Pipeline Integration
The complete driving-first pipeline is now:
1. **Waymo Episodes** → `training/rl/waymo_episode_dataset.py`
2. **SSL Pretrain** → `training/pretrain/ssl_pretrain.py` (NEW)
3. **Waypoint BC (SFT)** → `training/rl/proper_sft_rl_pipeline.py`
4. **RL Refinement** → `training/rl/grpo_waypoint.py`
5. **CARLA Eval** → `training/rl/carla_sft_rl_eval.py`
