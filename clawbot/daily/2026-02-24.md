## Pipeline PR #5: PPO Residual Delta Training for Waypoint RL After SFT

### Summary
Implemented Option B (action space = waypoint deltas) for RL refinement after SFT. This completes the RL-after-SFT pipeline with a clean implementation of residual delta learning.

### Key Insight
The residual delta pattern: `final_waypoints = sft_waypoints + delta_head(state)` allows RL to refine SFT predictions while keeping the baseline intact. The delta head learns to correct the SFT model based on environment feedback.

### Changes

#### `training/rl/waypoint_rl_env.py`
New Gym-style environment for waypoint RL:
- **State**: position (x, y), velocity (vx, vy), goal (goal_x, goal_y), SFT waypoints (H × 2)
- **Action**: delta waypoints (H × 2) - adjustments to SFT predictions
- **Dynamics**: Accelerate toward first waypoint, update SFT baseline each step
- **Reward**: goal proximity + smoothness penalty + collision penalty
- **Goal threshold**: 2.0m from target

#### `training/rl/ppo_residual_delta_train.py`
Full PPO training implementation:
- **SFTWaypointModel**: Frozen baseline predictor (MLP)
- **DeltaWaypointHead**: Trainable residual with Tanh bounded output (±2.0)
- **ValueFunction**: Advantage estimation
- **PPO Features**:
  - GAE (Generalized Advantage Estimation)
  - PPO clipped objective
  - Entropy bonus for exploration
  - KL regularization between SFT and delta
- **Architecture**: `final_waypoints = sft_waypoints + delta_head(z)`
