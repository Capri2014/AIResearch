## Pipeline PR #3: Multi-Seed Training & Trajectory Logging (2026-02-27)

### Summary
Added multi-seed training runner and trajectory logging utilities for comprehensive RL training analysis.

### Changes

**multi_seed_train.py** - Multi-seed training runner
- Run same training config with multiple random seeds
- Aggregate metrics across seeds (mean, std, min, max)
- Find robust checkpoints by mean performance
- Generate markdown reports with per-seed results
- CLI args: --seeds, --episodes, --parallel, --metric
- Usage: `python multi_seed_train.py --seeds 42 43 44 --episodes 200`

**trajectory_logger.py** - Episode trajectory logging
- Record detailed trajectories: states, actions, rewards
- Compare SFT vs RL waypoints per episode
- Track goal reach, collisions, timeouts
- TrajectoryAnalyzer for post-hoc analysis
- Failure mode categorization
- CLI: `python trajectory_logger.py --dir out/trajectories --report`

### Benefits
- Measure training variance and stability
- Find robust checkpoints across seeds
- Debug training issues with full trajectories
- Analyze failure modes systematically

## Pipeline PR #2: LR Warmup & Early Stopping (2026-02-27)

### Summary
Added learning rate warmup and early stopping utilities to `ppo_residual_delta_train.py` for more stable RL training.

### Changes

**LearningRateWarmup**
- Linear LR increase from `warmup_ratio * lr` to `lr` over `warmup_episodes`
- CLI args: `--warmup-episodes`, `--warmup-ratio`
- Useful for preventing early training instability

**EarlyStopping**
- Monitor training metrics (reward/goal_rate/entropy)
- Stop when improvement stalls (patience) or gradient explosion detected
- CLI args: `--early-stopping-patience`, `--early-stopping-metric`
- Prevents overfitting and wasted compute
