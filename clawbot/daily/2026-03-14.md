**Pipeline PR #1** | **Day:** Saturday | **Time:** 5:30am PT

## Focus
Add speed prediction to waypoint BC model - predicting appropriate speeds for each waypoint in the trajectory.

## Tasks Completed

### 1. Speed Prediction Module
- **Created:** `training/waypoint_speed_head.py`

**Key components:**
- **SpeedHead**: MLP for predicting speed at each waypoint timestep
- **SpeedHeadConfig**: Configuration with min/max speed bounds (0-15 m/s)
- **forward_with_waypoints()**: Speed conditioned on waypoint geometry
  - Uses curvature (direction changes) to reduce speed in turns
- **speed_l1_loss()** and **speed_mse_loss()**: Training losses
- **WaypointSpeedPolicy**: Combined waypoint + speed prediction wrapper
- **SpeedBCDataset**: Dataset for speed prediction training

### 2. CARLA Policy Wrapper Integration
- **Updated:** `sim/driving/carla_srunner/policy_wrapper.py`

**Key additions:**
- `waypoints_to_control()` now accepts `target_speeds` parameter
- `predict_with_speed()` for joint waypoint + speed prediction
- `_compute_speed_profile()` for geometry-based speed estimation
- Speed-aware throttle/brake control based on current vs target speed

## Architecture
```
Waypoint BC Model
         ↓
    waypoints (H, 2)
         ↓
SpeedHead(z, waypoints) → speeds (H,)
         ↓
Speed-aware control: throttle/brake based on target_speed vs current_speed
```
