## Pipeline PR #5: PPO SFT Delta - RL Refinement After BC

### Created: `training/rl/run_ppo_sft_delta.py`
- **PPOSFTDeltaConfig**: Configuration dataclass for PPO residual delta learning
- **PPOBCDeltaAgent**: PPO agent that learns residual delta-waypoint corrections on top of BC predictions
- **MockBCWaypointModel**: Fallback mock BC model when no checkpoint available
- Integrates with BCToRLBridge for loading trained BC checkpoints
- Tracks ADE/FDE metrics during training and evaluation

### Features:
- RL refinement AFTER SFT (waypoint policy), Option B: action space = waypoint deltas
- Loads BC checkpoint via BCToRLBridge (auto-detects latest if not provided)
- Learns residual corrections: final_waypoints = bc_waypoints + delta
- ADE/FDE metrics computation for evaluation
- Outputs `metrics.json` and `train_metrics.json` to `out/ppo_sft_delta/run_YYYY-MM-DD_HH-MM-SS/`

### Architecture:
```
State [4] → BC Model (or Mock)
                ↓
         BC Waypoints [8, 2]
                ↓
         + Delta [8, 2] (learned)
                ↓
         Final Waypoints [8, 2]
                ↓
         ToyWaypointEnv.step()
```

### Usage:
```bash
# Train with auto-detected BC checkpoint
python -m training.rl.run_ppo_sft_delta --num_episodes 50

# Train with specific BC checkpoint
python -m training.rl.run_ppo_sft_delta --bc_checkpoint out/waypoint_bc/run_XXXX/best.pt

# Train with mock BC (no checkpoint)
python -m training.rl.run_ppo_sft_delta --use_mock_bc --num_episodes 50

# With custom eval interval
python -m training.rl.run_ppo_sft_delta --num_episodes 100 --eval_interval 10
```

### Smoke Test Results (10 episodes, mock BC):

- ✓ ADE: ~27-33m, FDE: ~32-35m (expected for random policy)

## Pipeline PR #4: BEV Encoder Integration for Waypoint BC (This PR)

### Created: `training/bc/bev_encoder_integration.py`
- **BEVBCConfig**: Configuration dataclass for BEV-integrated BC training
- **BEVWaypointBCModel**: Full BC model with BEV encoder integration
- **create_bev_bc_model()**: Factory function to create model
- **find_latest_bev_encoder_checkpoint()**: Helper for checkpoint discovery

### Features:
- Supports camera + LiDAR based BEV encoding via sim/driving/carla_srunner
- Falls back to simple SSL encoder when BEV encoder unavailable
- Integrates with existing waypoint prediction head
- Dynamic BEV channel detection for flexible projection

### Architecture:
```
Camera Images → CameraToBEV → Camera BEV Features
LiDAR Points → LidarToBEV → LiDAR BEV Features
                              ↓
                      Feature Fusion
                              ↓
              BEV Features [B, channels, H, W]
                              ↓
                      bev_projection (AdaptiveAvgPool2d + FC)
                              ↓
                      Features [B, encoder_dim]
                              ↓
                              WaypointHead
                              ↓
              waypoints [B, num_waypoints, 2], speed [B, 1]
```

### Usage:
```python
from training.bc import BEVBCConfig, create_bev_bc_model

# With BEV encoder (camera + LiDAR)
config = BEVBCConfig(
    use_bev_encoder=True,
    input_types=["camera", "lidar"],
    encoder_dim=256,
    num_waypoints=8
)
model = create_bev_bc_model(config)

# Forward pass with BEV input
cameras = [torch.randn(2, 3, 128, 128)]  # front camera
lidar = torch.randn(2, 1000, 3)  # point cloud
waypoints, speed = model(x=None, cameras=cameras, lidar_points=lidar)

# With SSL encoder fallback (simple BEV image)
model_ssl = create_bev_bc_model(BEVBCConfig(use_bev_encoder=False))
waypoints, speed = model_ssl(bev_image)
```

### Smoke Test Results:
- ✓ SSL encoder mode (fallback): waypoints=torch.Size([2, 8, 2]), speed=torch.Size([2, 1])
- ✓ BEV encoder mode: waypoints=torch.Size([2, 8, 2]), speed=torch.Size([2, 1])

## Pipeline PR #2: BC Evaluation Module with ADE/FDE Metrics (This PR)

### Created: `training/bc/evaluate_bc.py`
- Evaluates trained Waypoint BC models
- Computes metrics:
  - **ADE** (Average Displacement Error) - mean L2 error across all waypoints
  - **FDE** (Final Displacement Error) - L2 error at final waypoint
  - **Success Rate** - percentage with FDE < 2m threshold
  - **Speed Error** - mean absolute speed prediction error
- Auto-detects latest BC checkpoint from `out/waypoint_bc/`
- Integrates with SSL encoder from PR #1
- Outputs `metrics.json` compatible with pipeline schema

### Usage:

# Evaluate specific checkpoint

## Pipeline PR #3: BC-to-RL Bridge Module (This PR)

### Created: `training/bc/bc_to_rl_bridge.py`
- Loads trained Waypoint BC model checkpoints
- Provides waypoint predictions as initial proposals for RL refinement
- Bridges BC training → RL refinement pipeline
- Includes:
  - `BCToRLBridge`: Main bridge class
  - `BCToRLBridgeConfig`: Configuration dataclass
  - `find_latest_bc_checkpoint()`: Helper to find latest BC checkpoint
  - `encode_state()`: Converts environment state to BC input format

### Features:
- Loads BC checkpoints (auto-detects latest from `out/waypoint_bc/`)
- Predicts waypoints from BEV input: `predict_waypoints(bev)` → `[B, 8, 2]`
- Predicts waypoints + speed: `predict(bev)` → `(waypoints, speed)`
- State encoding for toy environments: `encode_state(state)` → `[B, 3, 128, 128]`
- Delta head factory for residual learning: `create_delta_head(input_dim)`

### Usage:
```python
from training.bc import BCToRLBridge, find_latest_bc_checkpoint

# Load latest BC model
bridge = BCToRLBridge(checkpoint_path=find_latest_bc_checkpoint())

# Predict waypoints from BEV
bev_input = torch.randn(2, 3, 128, 128)
waypoints = bridge.predict_waypoints(bev_input)  # [2, 8, 2]

# Use in RL environment
state = env.reset()  # [state_dim]
bev = bridge.encode_state(state)
proposal_waypoints = bridge.predict_waypoints(bev)
env.set_proposal_waypoints(proposal_waypoints)
```

### Smoke Test Results:
- ✓ BCToRLBridge created without checkpoint
- ✓ State encoding: torch.Size([2, 6]) -> torch.Size([2, 3, 128, 128])
- ✓ Loaded BC model from checkpoint
- ✓ Waypoint prediction: torch.Size([2, 8, 2])
- ✓ Speed prediction: torch.Size([2, 1])
- ✓ Delta head: torch.Size([2, 256]) -> torch.Size([2, 2])
