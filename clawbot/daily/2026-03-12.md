## Pipeline PR #1: SSL Encoder Integration for Waypoint BC (This PR)

### Updated: `training/bc/waypoint_bc.py`
- Implemented `load_encoder()` method in WaypointBCModel to load pre-trained SSL encoder checkpoints
- Supports multiple checkpoint formats:
  - Direct encoder state_dict
  - TinyMultiCamEncoder format from training/pretrain/
- Added `freeze_encoder` config option to freeze SSL encoder weights during BC training
- Updated BCConfig with freeze_encoder flag (default: True)

### Updated: `training/bc/run_bc_train.py`
- Added `--freeze-encoder` flag (enabled by default)
- Added `find_latest_ssl_encoder()` helper to auto-detect latest SSL encoder checkpoint
- Auto-detects SSL encoder if `--ssl-encoder-path` not provided
- Searches in: out/pretrain_ssl_stub/, out/pretrain_*/

### Usage:
```bash
# Train with auto-detected SSL encoder (recommended)
python -m training.bc.run_bc_train --epochs 50 --batch-size 64

# Train with specific SSL encoder
python -m training.bc.run_bc_train --ssl-encoder-path out/pretrain_ssl_stub/encoder.pt

# Train from scratch (no SSL)
python -m training.bc.run_bc_train --epochs 50 --ssl-encoder-path=""
```

### Architecture:
```
BEV Input [B, 3, H, W]
    ↓
SSLEncoder.load_encoder(path) → loads checkpoint
    ↓
Features [B, encoder_dim] (frozen if freeze_encoder=True)
    ↓
WaypointHead
    ↓
waypoints [B, num_waypoints, 2], speed [B, 1]
```

## Pipeline PR #2: BC Evaluation Module with ADE/FDE Metrics (This PR)

### Created: `training/bc/evaluate_bc.py`
- Evaluates trained Waypoint BC models
- Computes metrics:
  - **ADE** (Average Displacement Error) - mean L2 error across all waypoints
  - **FDE** (Final Displacement Error) - L2 error at final waypoint
  - **Success Rate** - percentage with FDE < 2m threshold
  - **Speed Error** - mean absolute speed prediction error
- Auto-detects latest BC checkpoint from `out/waypoint_bc/`
- Integrates with SSL encoder from PR #1
- Outputs `metrics.json` compatible with pipeline schema

### Usage:
```bash
# Evaluate latest BC checkpoint
python -m training.bc.evaluate_bc --num-samples 1000

# Evaluate specific checkpoint
python -m training.bc.evaluate_bc --checkpoint out/waypoint_bc/run_XXXX/best.pt

# Validate model loading only
python -m training.bc.evaluate_bc --dry-run
```

### Evaluation Results (200 synthetic samples):
```

```

## Pipeline PR #3: BC-to-RL Bridge Module (This PR)

### Created: `training/bc/bc_to_rl_bridge.py`
- Loads trained Waypoint BC model checkpoints
- Provides waypoint predictions as initial proposals for RL refinement
- Bridges BC training → RL refinement pipeline
- Includes:
  - `BCToRLBridge`: Main bridge class
  - `BCToRLBridgeConfig`: Configuration dataclass
  - `find_latest_bc_checkpoint()`: Helper to find latest BC checkpoint
  - `encode_state()`: Converts environment state to BC input format

### Features:
- Loads BC checkpoints (auto-detects latest from `out/waypoint_bc/`)
- Predicts waypoints from BEV input: `predict_waypoints(bev)` → `[B, 8, 2]`
- Predicts waypoints + speed: `predict(bev)` → `(waypoints, speed)`
- State encoding for toy environments: `encode_state(state)` → `[B, 3, 128, 128]`
- Delta head factory for residual learning: `create_delta_head(input_dim)`

### Usage:
```python
from training.bc import BCToRLBridge, find_latest_bc_checkpoint

# Load latest BC model
bridge = BCToRLBridge(checkpoint_path=find_latest_bc_checkpoint())

# Predict waypoints from BEV
bev_input = torch.randn(2, 3, 128, 128)
waypoints = bridge.predict_waypoints(bev_input)  # [2, 8, 2]

# Use in RL environment
state = env.reset()  # [state_dim]
bev = bridge.encode_state(state)
proposal_waypoints = bridge.predict_waypoints(bev)
env.set_proposal_waypoints(proposal_waypoints)
```

### Smoke Test Results:
- ✓ BCToRLBridge created without checkpoint
- ✓ State encoding: torch.Size([2, 6]) -> torch.Size([2, 3, 128, 128])
- ✓ Loaded BC model from checkpoint
- ✓ Waypoint prediction: torch.Size([2, 8, 2])
- ✓ Speed prediction: torch.Size([2, 1])
- ✓ Delta head: torch.Size([2, 256]) -> torch.Size([2, 2])
