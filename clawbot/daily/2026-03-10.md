### Focus
Multi-scenario evaluation framework for kinematic waypoint environment

### Work Done

**Created: `training/rl/eval_multi_scenario.py`**
- Multi-scenario evaluation runner for kinematic waypoint RL environment
- Supports 5 scenario configurations: simple, moderate, hard, urban, highway
- Each scenario has unique: world_size, num_waypoints, max_steps, obstacle_density
- Aggregates ADE/FDE, route_completion, collisions, offroad metrics
- Outputs metrics.json compatible with data/schema/metrics.json (domain="rl")
- Supports both SFT and RL policies with checkpoint loading
- Bridges kinematic environment evaluation with broader CARLA eval framework

### Scenario Configs
- **simple**: 50x50 world, 5 waypoints, low obstacle density
- **moderate**: 80x80 world, 8 waypoints, medium density  
- **hard**: 120x120 world, 12 waypoints, high density
- **urban**: 100x100 world, 10 waypoints, intersections
- **highway**: 200x200 world, 6 waypoints, high speed

### Usage
```bash
# Evaluate SFT baseline across all scenarios
python -m training.rl.eval_multi_scenario \
    --policy sft \
    --scenarios simple,moderate,hard \
    --episodes 10

# Evaluate RL policy with checkpoint
python -m training.rl.eval_multi_scenario \
    --policy rl \
    --checkpoint out/rl_delta_waypoint_v0/best_reward.pt \
    --scenarios all \
    --episodes 20
```
