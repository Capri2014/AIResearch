## Pipeline PR #4: Image Encoder for Scene Transformer

### Completed
- **Created `training/sft/image_encoder.py`**
  - ImageEncoder: ViT-based encoder for camera/BEV images (5.05M params)
  - BEVEncoder: CNN backbone for efficient BEV feature extraction (1.75M params)
  - HybridImageEncoder: CNN + Transformer hybrid (9.67M params)
  - ImageSceneFusion: Fuse image features with vector-based scene encoder
  - Supports multiple fusion strategies: cross_attention, concatenation, add

### Architecture
```
Image Input (H x W x 3)
    ├── ImageEncoder (ViT): patch embeddings + CLS token
    ├── BEVEncoder (CNN): global BEV features
    └── HybridImageEncoder: CNN backbone + Transformer refinement

ImageSceneFusion:
    ├── cross_attention: Query vector features with image features
    ├── concatenation: Concat image and vector features
    └── add: Element-wise addition
```

### Enables
- Full e2e pipeline from camera/BEV images to waypoint prediction
- Fusion with existing vector-based scene encoder
- Hybrid sensor fusion (cameras + LiDAR vectors)

### Completed

  - SceneTransformerEncoder: Core encoder with temporal attention for agents
  - MapPolylineEncoder: Encodes map polylines (roads, lanes)
  - CrossAttentionLayer: Agent-to-map cross attention
  - AgentHistoryEncoder: Temporal self-attention over trajectory history
  - SceneTransformerWithWaypointHead: End-to-end model with waypoint prediction

### Architecture
```
SceneTransformerEncoder:

├── Agent-to-Map Cross Attention (num_layers=3)
├── Agent-to-Agent Self Attention
└── Output: agent_embeddings + scene_embedding

SceneTransformerWithWaypointHead:
├── SceneTransformerEncoder
└── ProposalWaypointHead (K=5 modes) or regression head
```
