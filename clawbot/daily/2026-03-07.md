# Daily Notes: 2026-03-07

## Pipeline PR #3 (This PR)

**Focus**: CARLA Evaluation Integration - Connect RL-to-CARLA pipeline with real CARLA evaluation

### What was done

Implemented actual CARLA evaluation in `rl_to_carla_pipeline.py`:

**Added functions:**
1. **`run_carla_evaluation()`** - Main CARLA evaluation function
   - Connects to CARLA server via Python client
   - Loads the specified map
   - Sets up sensors (RGB camera, collision sensor, lane invasion sensor)
   - Runs scenarios from the specified suite
   - Collects route_completion, success_rate, collisions, ADE/FDE

2. **`_get_scenario_list()`** - Returns scenarios based on suite
   - smoke: straight_clear, turn_left, turn_right
   - basic: adds cloudy, night variants
   - full: adds rain, lane_change, intersection

3. **`_setup_sensors()`** - Sets up vehicle sensors
   - RGB front-facing camera (512x256, 90° FOV)
   - Collision sensor for detecting impacts
   - Lane invasion sensor for detecting lane violations

4. **`_run_single_scenario()`** - Runs individual scenario
   - Spawns ego vehicle at spawn point
   - Runs policy loop with waypoint prediction
   - Tracks collisions and route completion

5. **`_compute_delta_magnitude()`** - Measures delta correction size
6. **`_compute_delta_effective()`** - Measures % of frames with non-zero delta

**Fixed:**
- Bug in main() where policy instantiation had malformed arguments

**Usage:**
```bash
# With CARLA server running
python -m training.rl.rl_to_carla_pipeline \
    --sft-checkpoint out/sft_waypoint_bc/model.pt \
    --rl-checkpoint out/kinematic_rl/best_delta.pt \
    --output-dir out/rl_carla_eval \
    --suite smoke

# Dry-run (no CARLA)
python -m training.rl.rl_to_carla_pipeline \
    --sft-checkpoint out/sft_waypoint_bc/model.pt \
    --rl-checkpoint out/kinematic_rl/best_delta.pt \
    --output-dir out/rl_carla_eval \
    --dry-run
```
