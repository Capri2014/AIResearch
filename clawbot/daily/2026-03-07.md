# Daily Notes: 2026-03-07

## Pipeline PR #5 (This PR)

**Focus**: RL refinement AFTER SFT (waypoint policy) - Option B: action space = waypoints / waypoint deltas

### What was done

Created `training/rl/kinematic_waypoint_env.py` - Kinematic Waypoint RL Environment:

**1. KinematicCar (Bicycle Model):**
- Realistic car kinematics with wheelbase, steering, speed
- Forward simulation with heading changes

**2. KinematicWaypointEnv:**
- Toy 2D environment that consumes predicted waypoints
- Option B design: `final_waypoints = sft_waypoints + delta_head(z)`
- Waypoint tracking with progress rewards
- Supports SFT checkpoint loading for Option B learning

**3. PPO Components:**
- `DeltaWaypointActor`: Actor predicting (H, 2) delta waypoints with exploration
- `ValueFunction`: Value estimation for PPO
- `PPOTrainer`: Complete PPO training with GAE advantages
- Delta bounds for safety (preserves SFT behavior)

**4. Training:**
- Run `python -m training.rl.kinematic_waypoint_env --episodes 100 --out-dir out/kinematic_rl/run_001`
- Saves `metrics.json` and `train_metrics.json` to output directory
- Best checkpoint tracking by reward

**Smoke test results:**
- Mean reward: 128.63 ± 30.72 (5 episodes)
- Success rate: 20%

**Usage:**
```bash
# Run training
python -m training.rl.kinematic_waypoint_env \
    --episodes 300 \
    --seed 42 \
    --out-dir out/kinematic_rl/run_001

# With SFT checkpoint (Option B initialization)
python -m training.rl.kinematic_waypoint_env \
    --sft-checkpoint out/waypoint_bc/model.pt \
    --episodes 300 \
    --out-dir out/kinematic_rl_with_sft

# Evaluation only
python -m training.rl.kinematic_waypoint_env --eval-only --seed 42
```

**Pipeline connection:**
```
Waymo episodes → SSL pretrain → waypoint BC → RL refinement → CARLA eval
                                              ↑
                          (this PR - kinematic env for RL)
```

## Pipeline PR #4 (Previous)

**Focus**: Waypoint BC Training Runner + RL Integration Pipeline

### What was done

Created two new modules for the driving-first pipeline:

**1. `training/data/train_waypoint_bc_runner.py`** - Training runner with checkpoint management:
- `WaypointBCTrainer` class: Complete training lifecycle management
- Checkpoint saving/loading with best-model tracking (best_loss.pt, best_ade.pt)
- Training history logging to JSON
- Early stopping with configurable patience
- ADE/FDE metrics computation
- Mixed precision training (AMP) support
- Export for RL refinement functionality

**Key features:**
- L1 + L2 + final-waypoint combined loss
- Best model tracking by both loss and ADE
- Learning rate scheduling (CosineAnnealingLR)
- Gradient clipping for stability
- Periodic and best checkpoint saving
- RL export metadata generation

**2. `training/data/waypoint_bc_to_rl.py`** - SFT to RL pipeline bridge:
- `WaypointBCToRLExporter`: Exports SFT checkpoints for RL delta-waypoint training
- `SFTtoRLPipeline`: Complete pipeline orchestrator
- Checkpoint comparison utilities

**Key features:**
- Extracts model state for RL training
- Freeze encoder configuration for transfer learning
- Generates RL training commands
- Checkpoint comparison by ADE/FDE/loss
- Pipeline config persistence

**Usage:**
```bash
# Train waypoint BC
python -m training.data.train_waypoint_bc_runner \
    --data-dir data/waymo \
    --output-dir out/waypoint_bc \
    --num-epochs 100 \
    --batch-size 32

# Export for RL
python -m training.data.waypoint_bc_to_rl \
    --mode export \
    --checkpoint out/waypoint_bc/checkpoints/best_ade.pt \
    --output out/sft_to_rl/best_model.pt

# Full SFT → RL pipeline
python -m training.data.waypoint_bc_to_rl \
    --mode full-pipeline \
    --sft-checkpoint out/waypoint_bc/checkpoints/best_ade.pt \
    --output-dir out/sft_to_rl
```

**Pipeline connection:**
```
Waymo episodes → SSL pretrain → waypoint BC → RL refinement → CARLA eval
                                              ↑
                          (this PR - exports for RL)
```
