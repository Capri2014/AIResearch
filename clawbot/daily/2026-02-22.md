## Pipeline PR #5: ResAD (Residual with Attention and Dynamics)

### Summary
Added ResAD implementation for enhanced residual delta-waypoint learning with attention, dynamics modeling, and uncertainty estimation. This builds on the basic PPO residual approach by adding temporal modeling and model-based RL capabilities.

### Commits
- `65dc135` - feat(rl): Add ResAD (Residual with Attention and Dynamics)

### New Files
- `training/rl/resad.py` - ResAD core implementation
- `training/rl/resad_train.py` - Training script
- `out/resad_waypoint/<run_id>/` - Training artifacts

### Key Components

#### WaypointAttention
- Multi-head attention (4 heads) for temporal waypoint dependencies
- Projects waypoints to hidden_dim, applies self-attention, projects back
- Residual connection with layer normalization for stable training

#### DynamicsModel
- Forward dynamics: predicts next_state = f(state, action)
- Enables model-based RL planning without environment interaction
- Critical for autonomous driving where real-world interaction is expensive

#### UncertaintyHead
- Outputs log variance for Monte Carlo dropout-style uncertainty
- NLL loss for training: 0.5 * log(sigma^2) + 0.5 * (target - pred)^2 / sigma^2
- Higher uncertainty = delta head is uncertain about predictions

### Architecture
```
ResADAgent:
├── SFTWaypointModel (frozen)
├── ResADResidualHead:
│   ├── WaypointAttention (temporal modeling)
│   ├── DynamicsModel (state prediction)
│   └── UncertaintyHead (confidence estimation)
├── ValueFunction
└── InertialReferenceTransform (world→vehicle frame)

Formula: final_waypoints = sft_waypoints + attention_dynamics_delta(z)
```

### Key Classes
- **WaypointAttention**: Multi-head self-attention for waypoint sequences
- **DynamicsModel**: Forward dynamics for model-based RL
- **UncertaintyHead**: NLL-based uncertainty estimation
- **ResADResidualHead**: Combines attention + dynamics + uncertainty
- **ResADAgent**: Complete agent with frozen SFT + trainable delta

### Parameters
| Parameter | Default | Description |
|-----------|---------|-------------|
| use_attention | True | Enable attention mechanism |
| use_dynamics | True | Enable dynamics model |
| use_uncertainty | True | Enable uncertainty estimation |
| dynamics_loss_weight | 0.1 | Weight for dynamics loss |

### Training Losses
1. **Policy Loss**: MSE to target delta (SFT - action)
2. **Value Loss**: MSE to returns
3. **Dynamics Loss**: MSE between predicted and actual next states
4. **Uncertainty Loss**: NLL for variance prediction
5. **KL Divergence**: Keep RL close to SFT baseline
6. **Entropy Bonus**: Encourage exploration

### Usage
```bash
# Train ResAD
python -m training.rl.resad_train \
    --episodes 100 \
    --horizon 20 \
    --hidden-dim 64 \
    --lr 0.0003 \
    --out-dir out/resad_waypoint
```

### Training Results (30 episodes)

- Final goal rate: 0%
- Final avg length: 100 steps (max)
- Policy loss: 0.010
- Dynamics loss: 2.08

### Why This Matters
1. **Temporal Modeling**: Attention captures dependencies between waypoints
2. **Model-Based RL**: Dynamics model enables planning without environment interaction
3. **Uncertainty**: Quantifies confidence for safe RL deployment
4. **Modular**: Each component can be enabled/disabled independently
5. **SFT-Compatible**: Works with frozen SFT baseline via residual learning

## Pipeline PR #4: Enhanced PPO with Value Function Improvements

### Summary
Added enhanced PPO implementation with value function improvements for more stable RL training. Key improvements include value clipping, Huber loss, value normalization, and GAE lambda sweep capability.

### Commits
- `b46bb5e` - feat(rl): Add enhanced PPO with value function improvements

### New Files
- `training/rl/enhanced_ppo_residual.py` - Enhanced PPO with value improvements

### Key Features
1. **Value Clipping**: PPO-style clipping to prevent large value updates
2. **Huber Loss**: More robust to outliers than MSE (less sensitive to extreme rewards)
3. **Value Normalization**: Running mean/std for stable learning
4. **GAE Lambda Sweep**: Configurable advantage estimation (start MC-like, become TD-like)
5. **Gradient Clipping**: Prevents exploding gradients
6. **Enhanced Value Head**: Skip connections for better gradient flow

### Architecture
```
EnhancedPPOResidualWaypointAgent:
├── SFTWaypointModel (frozen)
├── DeltaWaypointHead (trainable)
├── EnhancedValueHead (with clipping + skip)
└── ValueNormalizer (optional)
```

### Key Classes
- **ValueNormalizer**: Running statistics for value normalization
- **EnhancedValueHead**: Value function with clipping and skip connections
- **EnhancedPPOResidualWaypointAgent**: Main agent with all improvements

### Parameters
| Parameter | Default | Description |
|-----------|---------|-------------|
| use_value_clipping | True | PPO-style value clipping |
| use_value_norm | True | Running value normalization |
| use_huber_loss | True | Huber loss instead of MSE |
| huber_delta | 1.0 | Huber loss delta parameter |

### GAE Lambda Schedule
```python
gae_schedule = {
    0: 0.98,    # High lambda = Monte Carlo-like (low variance)
    50: 0.95,   # Mid lambda
    100: 0.90   # Lower lambda = TD-like (fine-grained)
}
```

### Usage
```python
agent = EnhancedPPOResidualWaypointAgent(
    state_dim=6,
    horizon=20,
    use_value_clipping=True,
    use_value_norm=True,
    use_huber_loss=True,
    huber_delta=1.0
)

# Train with GAE lambda schedule
metrics = train_enhanced_ppo_residual(
    env, agent,
    gae_lam_schedule={0: 0.98, 50: 0.95, 100: 0.90}
)
```

### Why This Matters
1. **Stability**: Value clipping + gradient clipping prevents training divergence
2. **Robustness**: Huber loss handles extreme rewards better than MSE
3. **Efficiency**: Value normalization enables faster learning
4. **Flexibility**: GAE lambda sweep allows curriculum-style training

## Pipeline PR #2: KL Regularization for PPO Residual Delta-Waypoint Training

### Summary
Added KL divergence regularization to keep the RL policy close to the frozen SFT baseline. This prevents the delta head from drifting too far from SFT predictions, improving training stability and safety.

### Commits
- `19bfb47` - feat(rl): Add KL regularization to PPO residual delta-waypoint training

### Changes
- Added `kl_coef` parameter (default 0.1) to `PPOResidualWaypointAgent`
- Added `compute_kl_divergence()` method: MSE-based KL approximation
- Updated loss to include `kl_loss = kl_coef * kl_div`
- Added `kl_divs` tracking in training metrics
- Logs KL divergence every 10 episodes

### Architecture
```
Loss = policy_loss + value_coef * value_loss + entropy_coef * entropy + kl_coef * KL(SFT || RL)
```

### Key Features
- **kl_coef** (default 0.1): Controls strength of KL regularization
- **MSE-based KL**: Approximates KL divergence using 0.5 * ||pred - sft||^2
- **Training monitoring**: Logs KL divergence to detect policy drift
- **Safe RL**: Keeps delta head close to SFT baseline

### Usage
```python
agent = PPOResidualWaypointAgent(
    state_dim=6,
    horizon=20,
    kl_coef=0.1,  # KL regularization coefficient
    use_residual=True
)
```

### Why This Matters
1. **Stability**: Prevents RL from deviating too far from SFT
2. **Safety**: Ensures RL improvements don't break SFT's good behaviors
3. **Interpretability**: KL divergence is a well-understood measure of policy change

## Pipeline PR #3: Multi-Scenario RL with Domain Randomization

### Summary
Added multi-scenario RL training with domain randomization for robust autonomous driving across different weather and lighting conditions. This is critical for sim-to-real transfer and handling real-world variability.

### Commits
- `ad9fa86` - feat(rl): Add multi-scenario RL with domain randomization for robust driving

### New Files
- `training/rl/multi_scenario_env.py` - Multi-scenario environment with 5 conditions
- `training/rl/train_multi_scenario_rl.py` - Training and evaluation scripts

### Scenario Types
- **clear**: Standard conditions (difficulty 1.0)
- **cloudy**: Reduced lighting, softer shadows (difficulty 1.2)
- **night**: Low visibility, artificial lighting (difficulty 1.5)
- **rain**: Reduced friction, wet surfaces (difficulty 1.8)
- **fog**: Reduced visibility, distance-dependent (difficulty 2.0)

### Key Features
- **Domain randomization**: Varies visibility (±20%), friction (±10%), noise (±50%) each episode
- **CurriculumScheduler**: Gradually increases scenario difficulty (cosine annealing)
- **Scenario-specific reward shaping**: Night penalizes speed, rain penalizes high velocity
- **Per-scenario evaluation**: Validates policy robustness across all conditions

### Architecture
```
MultiScenarioEnv → ResidualDeltaNetwork → final_waypoints = sft_waypoints + delta_head(z)
```

### Scenario Parameters
| Scenario | Visibility | Friction | Noise | Light Factor |
|----------|------------|----------|-------|--------------|
| clear    | 100.0      | 1.0      | 0.0   | 1.0          |
| cloudy   | 80.0       | 0.95     | 0.05  | 0.8          |
| night    | 50.0       | 0.9      | 0.15  | 0.3          |
| rain     | 40.0       | 0.7      | 0.2   | 0.6          |
| fog      | 30.0       | 0.85     | 0.25  | 0.5          |

### Usage
```bash
# Train with domain randomization
python -m training.rl.train_multi_scenario_rl \
```
