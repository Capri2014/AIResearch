## Pipeline PR #2: KL Regularization for PPO Residual Delta-Waypoint Training

### Summary
Added KL divergence regularization to keep the RL policy close to the frozen SFT baseline. This prevents the delta head from drifting too far from SFT predictions, improving training stability and safety.

### Commits
- `19bfb47` - feat(rl): Add KL regularization to PPO residual delta-waypoint training

### Changes
- Added `kl_coef` parameter (default 0.1) to `PPOResidualWaypointAgent`
- Added `compute_kl_divergence()` method: MSE-based KL approximation
- Updated loss to include `kl_loss = kl_coef * kl_div`
- Added `kl_divs` tracking in training metrics
- Logs KL divergence every 10 episodes

### Architecture
```
Loss = policy_loss + value_coef * value_loss + entropy_coef * entropy + kl_coef * KL(SFT || RL)
```

### Key Features
- **kl_coef** (default 0.1): Controls strength of KL regularization
- **MSE-based KL**: Approximates KL divergence using 0.5 * ||pred - sft||^2
- **Training monitoring**: Logs KL divergence to detect policy drift
- **Safe RL**: Keeps delta head close to SFT baseline

### Usage
```python
agent = PPOResidualWaypointAgent(
    state_dim=6,
    horizon=20,
    kl_coef=0.1,  # KL regularization coefficient
    use_residual=True
)
```

### Why This Matters
1. **Stability**: Prevents RL from deviating too far from SFT
2. **Safety**: Ensures RL improvements don't break SFT's good behaviors
3. **Interpretability**: KL divergence is a well-understood measure of policy change

## Pipeline PR #1: Offline vs Closed-Loop Metrics Correlation Analysis

### Summary
Added script to analyze the correlation between offline waypoint prediction metrics (ADE/FDE) and closed-loop CARLA metrics (route_completion, collision_rate). This helps validate that offline improvements translate to closed-loop performance and guides checkpoint selection.

### Commits
- `f327be0` - feat(eval): Add offline vs closed-loop metrics correlation analysis

### New Files
- `training/eval/correlate_offline_closed_loop.py` - Correlation analysis script

### Architecture
```
Offline Metrics (ADE/FDE) ←→ Closed-Loop Metrics (CARLA)
```

### Key Features
- **Pearson correlation**: Computes correlation between offline and closed-loop metrics
- **Domain-specific analysis**: Separates SFT, PPO, GRPO correlations
- **Interpretation**: Auto-generates human-readable interpretation of results
- **JSON output**: Machine-readable results for downstream analysis

### Usage
```bash
# Analyze correlation from existing eval directories
python -m training.eval.correlate_offline_closed_loop \
  --offline-dir out/eval/20260218-213206 \
  --carla-dir out/carla_closed_loop_eval \
  --output correlation_report.json \
  --print-report
```

### Output
- Correlation coefficients: ade_vs_route_completion, ade_vs_collision_rate, fde_vs_route_completion, fde_vs_collision_rate
- Interpretation of correlation strength and direction
- Domain-specific breakdowns

### Why This Matters
1. **Validates offline improvements**: Do lower ADE/FDE actually lead to better driving?
2. **Guides checkpoint selection**: Can we use offline metrics to pick best checkpoints?
3. **Identifies distribution shift**: When offline metrics are misleading
