## Pipeline PR #1: Auto Checkpoint Selection for CARLA Evaluation (8:30am PT)

**Date:** 2026-02-26
**Time:** 8:30 AM (America/New_York) / 5:30 AM (America/Los_Angeles)
**Branch:** feature/daily-2026-02-26-a
**Commit:** 7605c48

### Changes

Added automatic checkpoint selection to CARLA evaluation pipeline:

1. **New CLI Arguments**
   - `--auto-select`: Automatically select best checkpoint from training runs
   - `--select-criterion`: Choose selection criterion (reward, entropy, ade, fde, success)
   - `--select-domain`: Filter runs by domain (default: rl)

2. **auto_select_checkpoint() function**
   - Integrates with CheckpointManager to scan training runs
   - Supports multiple selection criteria
   - Falls back to finding checkpoint files in run directory

3. **Selection criteria**
   - `reward`: Highest final reward
   - `entropy`: Highest entropy (most exploratory)
   - `ade`: Best Average Displacement Error (lowest)
   - `fde`: Best Final Displacement Error (lowest)
   - `success`: Highest success rate

4. **Output integration**
   - Auto-selection info saved in metrics.json
   - Includes criterion, run_id, metrics, and checkpoint path

### Usage

```bash
# Auto-select best by reward
python carla_sft_rl_eval.py --auto-select --select-criterion reward

# Auto-select best by ADE
python carla_sft_rl_eval.py --auto-select --select-criterion ade

# Auto-select by entropy (most exploratory)
python carla_sft_rl_eval.py --auto-select --select-criterion entropy

# Manual checkpoint (still works)
python carla_sft_rl_eval.py --checkpoint path/to/checkpoint.pt
```

### Benefits

- **No manual checkpoint selection**: Automatically finds best checkpoint
- **Flexible criteria**: Choose based on reward, exploration, or metrics
- **Metrics-driven**: ADE/FDE/success integrated into selection
- **Reproducible**: Selection info saved in output

### Architecture Context

```
Training Runs (out/*/train_metrics.json)
        ↓
CheckpointManager + CheckpointSelector
        ↓
Auto-selected checkpoint → CARLA Evaluation
```

**Branch:** feature/daily-2026-02-26-b
**Commit:** 317f8a3

### Changes

Added gradient norm tracking to RL training for improved stability monitoring:

1. **ppo_residual_delta_train.py**
   - Track gradient norms per training update
   - Log average grad norm every 20 episodes
   - Include grad_norms in returned metrics

2. **grpo_waypoint.py**
   - Added gradient clipping with norm tracking (0.5 max)
   - Track grad_norm in update() return dict
   - Log grad norm in training progress output

3. **enhanced_ppo_residual.py**
   - Track delta_head and value_fn gradient norms separately
   - Sum for total grad norm tracking
   - Include in metrics and logging

### Benefits

- **Debugging**: Quickly identify training instability issues
- **Monitoring**: See when gradient clipping is actively reducing norms  
- **Analysis**: Compare delta head training stability vs full model
- **Checkpoint selection**: Can use gradient norm as additional criterion

### Usage

Gradient norms are automatically logged during training:
```
Episode 20: reward=..., grad_norm=0.45
Update 2/10: ..., Grad Norm: 0.32
```

Saved in train_metrics.json for later analysis.

## Pipeline PR #3: Training Visualization Utility (1:30pm PT)

**Date:** 2026-02-26
**Time:** 1:30 PM (America/New_York) / 10:30 AM (America/Los_Angeles)
**Branch:** feature/daily-2026-02-26-c
**Commit:** 803851b

### Changes

Added training visualization utility for analyzing training curves and evaluation metrics:

1. **visualize_training.py** - New comprehensive visualization script
   - **Training curves**: reward, entropy, gradient norm, loss over episodes
   - **Evaluation metrics**: ADE, FDE, success rate, average return
   - **Multi-run comparison**: Compare metrics across multiple checkpoints
   - **HTML reports**: Generate self-contained HTML reports with embedded plots

2. **Features**
   - Automatic metrics detection (train_metrics.json, metrics.json)
   - Moving average smoothing for noisy training curves
   - Bar charts for eval metrics comparison
   - Color-coded multi-run comparisons
   - CLI interface with flexible options

3. **CLI Options**
   - `--run`: Path to training run or eval directory
   - `--compare`: Multiple runs for comparison
   - `--plot`: Type of plot (training/eval/both)
   - `--metric`: Metric for comparison (ade/fde/success/return)
   - `--output`: Output file path
   - `--format`: Output format (png/pdf/html)

### Usage

```bash
# Plot training curves from a run
python training/rl/visualize_training.py --run out/rl_delta_toy_v0/<run_id> --plot training

# Plot evaluation metrics
python training/rl/visualize_training.py --run out/eval/<eval_id> --plot eval

# Compare multiple runs by ADE
python training/rl/visualize_training.py --compare run1 run2 --metric ade

# Generate HTML report
python training/rl/visualize_training.py --run <id> --output report.html
```

### Benefits

- **Training analysis**: Visualize reward/entropy/gradient curves over time
- **Eval comparison**: Compare ADE/FDE/success across checkpoints
- **Reporting**: Generate shareable HTML reports
- **Debugging**: Identify training issues from visualization
- **Multi-seed**: Compare runs across different random seeds

### Architecture Context

Visualization complements existing training and evaluation tools:
```
Training → train_metrics.json → visualize_training.py → plots/report
Eval → metrics.json → visualize_training.py → comparison plots
```
