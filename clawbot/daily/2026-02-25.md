## Pipeline PR #5: SFT Checkpoint Loading for Residual Delta Training (4:30pm PT)

**Date:** 2026-02-25
**Time:** 4:30 PM (America/Los_Angeles) / 7:30 PM (America/New_York)
**Branch:** feature/daily-2026-02-25-e

### Changes

Added ability to load pretrained SFT checkpoint for residual delta-waypoint training:

1. **load_sft_checkpoint() function** (`training/rl/ppo_residual_delta_train.py`)
   - Load pretrained SFT waypoint model from checkpoint file (.pt)
   - Supports 'model_state' and 'state_dict' keys
   - Automatically freezes parameters and sets to eval() mode
   - Falls back to random init if checkpoint not found

2. **PPOResidualDeltaAgent update**
   - Added optional `sft_model` parameter to accept pre-loaded SFT model
   - Enables initializing from trained SFT checkpoint instead of random init

3. **New CLI args**
   - `--sft-checkpoint <path>`: Path to SFT checkpoint file
   - `--lr`: Learning rate (default 3e-4)

### Benefits

- **RL after SFT**: Load frozen SFT model, train only residual delta head
- **Architecture**: final_waypoints = sft_waypoints + delta_head(state)
- **Checkpoint Reuse**: Enables loading any trained waypoint model
- **Flexibility**: Works with or without SFT checkpoint

### Usage

```bash
# Train with pretrained SFT checkpoint
python ppo_residual_delta_train.py --sft-checkpoint path/to/sft_checkpoint.pt

# Train from scratch (no checkpoint)
python ppo_residual_delta_train.py
```

## Pipeline PR #4: Multi-Run Comparison & Metric-Based Checkpoint Selection

**Date:** 2026-02-25
**Time:** 4:30 PM (America/New_York) / 1:30 PM (America/Los_Angeles)
**Branch:** feature/daily-2026-02-25-d

### Changes

Added multi-run comparison utilities for analyzing RL training across different seeds:

1. **MultiRunComparator class** (`training/rl/multi_run_compare.py`)
   - Scan and load metrics from multiple training runs
   - Track: reward, entropy, ADE, FDE, success_rate, goal_reach_rate
   - Load from train_metrics.json and eval_metrics.json

2. **Best-by-Metric Selection**
   - `--best reward`: Highest reward checkpoint
   - `--best entropy`: Most exploratory (highest entropy)
   - `--best ade`: Best trajectory accuracy (lowest ADE)
   - `--best fde`: Best final position accuracy (lowest FDE)
   - `--best success`: Highest success rate

3. **CLI Interface**
   - `--list`: List all runs with key metrics
   - `--compare <run_ids>`: Compare specific runs
   - `--report`: Generate markdown comparison report
   - `--domain`: Filter runs by domain (e.g., rl_delta)

### Benefits

- **Cross-Seed Analysis**: Compare runs with different random seeds
- **Metric Integration**: ADE/FDE from eval_metrics.py integrated into selection
- **Reporting**: Markdown reports for documentation
- **Decision Support**: Recommendations for which checkpoint to use
