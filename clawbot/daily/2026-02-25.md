## Pipeline PR #3: Checkpoint Manager for Training Run Comparison

**Date:** 2026-02-25
**Time:** 1:30 PM (America/New_York)
**Branch:** feature/daily-2026-02-25-c

### Changes

Added checkpoint management and selection utilities for RL training:

1. **CheckpointManager class** (`training/rl/checkpoint_manager.py`)
   - Load metrics from multiple training runs
   - Track final_reward, best_reward, entropy, ADE, FDE, success_rate
   - List runs with optional domain filtering

2. **CheckpointSelector class**
   - Select best checkpoint by various criteria: reward, entropy, ADE, FDE, success
   - Compare multiple runs side by side
   - Generate text comparison reports

3. **CLI interface**
   - `--list`: List all runs
   - `--report`: Generate comparison report
   - `--best <criterion>`: Select best checkpoint
   - `--compare <run_ids>`: Compare specific runs

### Benefits

- **Unified Interface**: Compare RL training runs across different seeds
- **Flexible Selection**: Choose best checkpoint by reward, exploration, or metrics
- **Progress Tracking**: Track training progress over time
- **ADE/FDE Integration**: Support for metric-based checkpoint selection

## Pipeline PR #1: Learning Rate Scheduling for RL Training (Morning)

**Date:** 2026-02-25
**Time:** 8:30 AM (America/New_York)
**Branch:** feature/daily-2026-02-25-a

### Changes

Added learning rate scheduling to improve RL training stability and convergence:

1. **PPO Delta-Waypoint Training** (`training/rl/train_rl_delta_waypoint.py`)
   - Added `lr_min` (1e-5) and `lr_warmup_epochs` (10) to PPOConfig
   - Implemented SequentialLR with LinearLR warmup + CosineAnnealingLR decay
   - Added LR logging to training metrics

2. **GRPO Delta-Waypoint Training** (`training/rl/train_grpo_delta_waypoint.py`)
   - Added same LR scheduling parameters
   - Added CLI args: `--delta-lr-min`, `--delta-lr-warmup-epochs`

### Benefits

- **Warmup**: Initial low LR (10% of target) prevents early training instability
- **Cosine Decay**: Smooth LR reduction from lr to lr_min over training
- **Logging**: Track LR throughout training for debugging
