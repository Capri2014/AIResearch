## Pipeline PR #4: Training Diagnostics Module

### Problem
Training RL policies is complex and prone to various failure modes (gradient explosion, reward collapse, entropy collapse, training plateaus). Debugging training issues requires manually inspecting metrics and understanding what went wrong.

### Solution
Added comprehensive training diagnostics module to automatically identify failure modes and provide actionable recommendations:

1. **training_diagnostics.py** (NEW - 620+ lines)
   - **TrainingDiagnosticsAnalyzer**: Analyzes training runs
   - **Failure Mode Detection**:
     - Gradient explosion (grad_norm > 100)
     - Reward collapse (all recent rewards ≤ 0)
     - Entropy collapse (policy became deterministic too early)
     - Training plateau (no improvement over 50 episodes)
     - Value function divergence (vloss > 100)
     - Oscillating rewards (high variance in rolling mean)
   - **Learning Dynamics Analysis**:
     - Reward trend detection (increasing, decreasing, stable, oscillating)
     - Entropy trend analysis (exploration behavior)
     - Value alignment (correlation between predicted and actual returns)
   - **Actionable Recommendations**: Generated based on detected issues

2. **CLI Interface**:
   - `--list`: Show all available training runs
   - `--latest`: Analyze most recent run
   - `--run-id <id>`: Analyze specific run
   - `--report`: Generate markdown report
   - `--failure-modes`: Show failure mode analysis

### Testing

```bash

```

## Pipeline PR #3: Advanced Checkpoint Selection with ADE/FDE Metrics

### Problem
Checkpoint selection in the driving pipeline was based on training reward, which doesn't directly correlate with driving quality. ADE (Average Displacement Error) and FDE (Final Displacement Error) are the standard metrics for driving precision, but weren't being used for checkpoint selection.

### Solution
Added driving-specific checkpoint selection based on evaluation metrics:

1. **checkpoint_selector_advanced.py** (NEW - 450+ lines)
   - `DrivingMetrics`: Dataclass for ADE, FDE, success rate, route completion, collisions
   - `AdvancedCheckpointSelector`: Select best checkpoints by:
     - ADE (lower is better)
     - FDE (lower is better)
     - Success rate (higher is better)
     - Composite score (configurable weights)
   - `EvaluationReporter`: Generate markdown reports
   - CLI interface with --best-ade, --best-fde, --best-composite, --report

2. **eval_quick.py** (Enhanced)
   - --show-checkpoints: Display available checkpoints with metrics
   - --best-ade/--best-fde/--best-success/--best-composite: Smart selection
   - Integration with AdvancedCheckpointSelector

### Testing

# Select best by ADE
python -m training.rl.checkpoint_selector_advanced --best-ade

# Generate markdown report
