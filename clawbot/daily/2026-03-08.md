### Pipeline PR #1: Trajectory Planning Interface
Created new `training/planning/` module to bridge waypoint predictions to CARLA evaluation:

**New files:**
- `training/planning/__init__.py` - Package init with exports
- `training/planning/trajectory_planner.py` - Main trajectory planning module

**Key components:**
- `TrajectoryPlanner`: Core planner class with:
  - Cubic spline and linear interpolation
  - Kinematic smoothing with acceleration constraints
  - Speed profile optimization
  - Heading computation from trajectory geometry
  - Batch planning support
  
- `TrajectoryPlannerConfig`: Configuration dataclass
- `Trajectory` / `TrajectoryPoint`: Data structures
- CARLA integration helpers:
  - `trajectory_to_carla_waypoints()`
  - `waypoints_to_carla_transforms()`

**Integration points:**
- Input: Waypoint BC outputs (numpy arrays)
- Output: Smooth trajectories for CARLA closed-loop eval
- Used by: `training/eval/run_carla_closed_loop_eval.py`
